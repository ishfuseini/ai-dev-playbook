# 🎨 Lightweight CSS / UI Frameworks

## Utility-First (Minimal + Fast)
- **Tailwind CSS** – Utility-first, highly flexible (can feel heavy but very productive)
- **UnoCSS** – On-demand atomic CSS (lighter, faster alternative to Tailwind)
- **Windi CSS** – Similar to Tailwind, optimized runtime *(note: largely superseded by UnoCSS)*

---

## Classless / Minimal Styling
Drop in and forget — works with plain semantic HTML, no class names required.

- **Pico.css** – Classless, clean defaults, just works with semantic HTML
- **Water.css** – Drop-in styling, zero classes
- **Simple.css** – Minimal, readable default styles

---

## Lightweight Component Libraries
Pre-built components built on top of utility frameworks.

- **daisyUI** – Tailwind-based, adds simple semantic component classes
- **Flowbite** – Tailwind components with interactivity
- **Preline UI** – Lightweight Tailwind component set

---

## Headless / Unstyled Primitives
Behaviour and accessibility only — bring your own styles.

- **Radix UI** – Unstyled, accessible primitives (pair with Tailwind or your own CSS)
- **shadcn/ui** – Copy-paste components built on Tailwind + Radix (not a dependency, you own the code)
- **Ark UI** – Headless UI components (framework-agnostic)

---

## Vanilla CSS Frameworks
Traditional class-based frameworks with no JavaScript dependency.

- **Milligram** – Tiny (~2kb), flexbox-based
- **Skeleton** – Very small boilerplate framework
- **Spectre.css** – Modern, lightweight components
- **Bulma** – Simple class-based framework, no JS
- **UIkit** – Modular and relatively lightweight
- **Chota** – Very small, modern CSS framework
