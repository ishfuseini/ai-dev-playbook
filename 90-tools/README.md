# 90 — Tools

A curated directory of external tools that complement AI-assisted development workflows. Each file covers a specific domain and lists tools with descriptions, links, and guidance on when to use them.

This section is a **reference**, not a prompt collection — use it to discover and compare tools, then reach for the relevant prompt files in other sections of the playbook to put them to work.

---

## Contents

| File | Purpose |
|------|---------|
| `development.md` | AI coding agents, code review, testing, local dev, CI/CD, and deployment tools |
| `documentation.md` | Tools for generating, publishing, and maintaining project documentation |
| `planning.md` | Spec creation, task management, roadmapping, and memory system tools |
| `security.md` | SAST, secret scanning, dependency auditing, and security review tools |

---

## How to Use

1. Browse the relevant file for your current need
2. Each tool includes a link and a short description of what it does and when to reach for it
3. Where tools overlap, guidance is provided on how to choose between them
